- ✅ Survives page refresh.
- ✅ Survives navigation.

- ✅ Quantity updates persist.

## Logged-in Cart
- ✅ Fetches from server on login.
- ✅ Add/update/remove/clear work via API.

## Stock Validation
- ✅ Cannot add out-of-stock products.
- ✅ Cannot exceed available stock when increasing quantity.
- ✅ Toast warning shown at stock limit.
- ✅ Low-stock and out-of-stock badges displayed.

## Edge Cases
- ✅ Duplicate additions increment quantity.
- ✅ Invalid quantity input is ignored.
- ✅ Quantity cannot go below 1.
- ✅ Removing last item shows empty state.
- ✅ Deleted/unavailable products show placeholder image and out-of-stock badge.

- Guest cart builds from localStorage synchronously; no extra API calls.
- Authenticated cart uses React Query with staleTime 0 for freshness.
- No unnecessary re-renders observed in context design.

## Known Limitations
