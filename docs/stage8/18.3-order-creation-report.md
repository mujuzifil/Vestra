# Stage 18.3 — Order Creation Report

## Order Lifecycle

```
Place Order
  ↓
Validate customer & shipping info (frontend)
  ↓
Auto-register guest if needed (frontend)
  ↓
Merge guest cart to server (frontend)
  ↓
POST /checkout
  ↓
Validate request (CheckoutRequest)
  ↓
Validate cart not empty (OrderService)
  ↓
Validate product status and stock (OrderService)
  ↓
Calculate subtotal, shipping, tax, total (OrderService)
  ↓
Create Order record
  ↓
Create OrderItem records
  ↓
Decrement stock for COD orders
  ↓
Initiate digital payment for non-COD orders
  ↓
Clear cart
  ↓
Return Order
  ↓
Redirect to /checkout/confirm?order={id}
```

## Validation

### Frontend
- Empty cart check before checkout.
- Customer name, email, phone validation.
- Shipping address validation.
- Payment method selection.
- Duplicate submission prevention with loading state.

### Backend
- `CheckoutRequest` validates payment method enum and shipping address fields.
- `OrderService` validates cart exists and is not empty.
- `OrderService` validates each product is active.
- `OrderService` validates sufficient stock.
- Totals are calculated server-side to prevent price manipulation.

## Order Numbering
- Invoice number format: `VST-YYYYMMDD-XXXX`
- Generated by `OrderService::generateInvoiceNumber()`.
- Sequenced per day.

## Totals Calculation
- **Subtotal**: sum of (quantity × unit price) for all items.
- **Shipping**: read from `shipping_cost` setting (default 0) OR frontend-provided shipping cost for express.
- **Tax**: subtotal × tax rate (default 18%).
- **Total**: subtotal + shipping + tax.

## Notes
- Digital payment initiation is already wired in `OrderService` but the frontend does not redirect yet.
