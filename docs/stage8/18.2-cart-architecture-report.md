# Stage 18.2 — Cart Architecture Report

## Existing Infrastructure (pre-stage)
- `frontend/lib/cart-context.tsx` — Context provider supporting guest localStorage cart and authenticated server cart.
- `frontend/lib/api/cart.ts` — API wrappers for get/add/update/remove/clear/merge.
- `frontend/components/cart/cart-drawer.tsx` — Slide-out mini cart.
- `frontend/app/checkout/checkout-page-client.tsx` — Existing checkout flow.

## What Already Worked
- Authenticated cart fetch, add, update, remove, clear.
- Merge-on-login flow.
- Cart drawer rendering for logged-in users.
- Cart count badge in navbar.

## What Was Incomplete
- Guest cart stored only `{ product_id, quantity }`, so the drawer rendered placeholder text, no images, and no prices.
- No toast/notification feedback for cart actions.
- No full `/cart` page.
- No stock validation client-side.
- Quantity controls were +/- only.
- Empty cart state was minimal.
- Cart badge had no animation.

## Improvements Made

### Guest cart enrichment
- Guest cart items now store a product snapshot: `id, name, slug, sku, price, stock_quantity, images`.
- The context builds a full `Cart` object for guests with line totals and item counts.

### Stock validation
- `addItem` checks product stock before adding.
- `updateItem` prevents exceeding available stock.
- Toast warning shown when stock limit is reached.

### Merge-on-login robustness
- Added a ref guard so the guest-to-server merge only runs once per login session.
- Added error handling with toast feedback.

### Type safety
- Added `CartItemProduct` subset type in `frontend/types/index.ts` so cart items do not need a full `Product` object.
- Added `toCartProduct()` helper to convert `Product` to `CartItemProduct`.
