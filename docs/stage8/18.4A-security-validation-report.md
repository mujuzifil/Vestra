# Stage 18.4A — Security Validation Report

## HTTPS Enforcement

- All three vhosts listen on 443 only.
- The default HTTP server returns 301 to HTTPS for all requests except ACME challenges.
- HSTS is enabled on the frontend and admin vhosts with:
  - `max-age=31536000`
  - `includeSubDomains`
  - `preload`

## TLS Configuration

Nginx `nginx.conf` already configures:

- TLS 1.2 and 1.3 only
- Modern cipher suite
- Session cache and timeout
- No session tickets

## Security Headers

The `SecurityHeaders` middleware already emits:

- `X-Frame-Options: DENY`
- `X-Content-Type-Options: nosniff`
- `Referrer-Policy: strict-origin-when-cross-origin`
- `Permissions-Policy: camera=(), microphone=(), geolocation=()`
- `Strict-Transport-Security` (production only)
- Content Security Policy

These headers apply to all Laravel responses, including the admin panel.

## Session Security

| Setting | Value | Purpose |
|---------|-------|---------|
| `SESSION_DRIVER` | `redis` | Session storage isolated from the filesystem |
| `SESSION_DOMAIN` | `.vestradetergents.com` | Cookie valid across subdomains |
| `SESSION_SECURE_COOKIE` | `true` | HTTPS-only cookies |
| `SESSION_SAME_SITE` | `strict` | CSRF mitigation |
| `SESSION_ENCRYPT` | `true` | Encrypted session data |

## Admin Isolation

- The admin panel now runs on a dedicated subdomain.
- The API domain no longer serves the admin panel; requests to `/admin` are redirected away.
- Filament auth middleware continues to require an authenticated admin user.
- `EnsureAdminPasswordChanged` enforces password-change policy for admins.

## CSRF Protection

- Filament forms include CSRF tokens via Laravel's `VerifyCsrfToken` middleware.
- `SESSION_SAME_SITE=strict` provides additional CSRF protection.
