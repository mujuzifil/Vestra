# Stage 18.4 — Customer Experience Report

## Checkout Payment Step

On the order confirmation page (`/checkout/confirm?order={id}`):

- Digital-payment orders display a clear "Proceed to Payment" call to action.
- Clicking the button calls `POST /payments/initiate` and immediately redirects to Flutterwave.
- A loading spinner disables the button during initiation.
- Errors display a toast notification.
- COD orders skip this step and show confirmation only.

## Supported Payment Methods

Flutterwave is configured to present the methods the customer selects at checkout:

| Checkout Selection | Flutterwave Method |
|--------------------|--------------------|
| MTN Mobile Money | Mobile Money |
| Airtel Money | Mobile Money |
| Card Payment | Visa / Mastercard |

The actual method availability is controlled by Flutterwave based on currency (`UGX`) and merchant configuration.

## Payment Return Page

URL: `/checkout/return?status={status}&tx_ref={ref}&transaction_id={id}`

### Successful Payment

- Shows "Payment Successful!" with a green checkmark.
- Displays the backend verification message.
- Provides CTAs:
  - View My Orders
  - Continue Shopping

### Cancelled Payment

- Shows "Payment Cancelled" with a red icon.
- Displays the transaction reference.
- Provides CTAs:
  - Retry Payment
  - View My Orders
  - Return to Checkout

### Failed Payment

- Shows "Payment Issue" with a red icon.
- Displays the failure reason from verification.
- Provides CTAs:
  - Retry Payment
  - View My Orders
  - Return to Checkout

### Pending Payment

- Shows "Payment Pending" with an amber alert icon.
- Informs the customer that the payment is still being processed.
- Provides CTAs:
  - Retry Payment
  - View My Orders
  - Return to Checkout

## Retry Payment

Customers can retry payment from two locations:

1. **Payment Return Page** — if verification returns a non-success state, a "Retry Payment" button re-initiates the same order's payment.
2. **Order Detail Page** — any order with `payment_status !== 'paid'` and a non-final order status shows a "Retry Payment" button.

Retry uses the existing `initiatePayment` flow, creating a new Flutterwave transaction reference without creating a duplicate order.

## Order History

The order list and order detail pages already display:

- Payment method
- Payment status (pending, paid, failed)
- Order status

After payment, customers see the updated status reflected on their next visit.

## Email Notifications

Email triggers are not implemented in this stage. The following notifications are recommended for Stage 18.5 or a dedicated notification stage:

- Order Created
- Payment Received
- Payment Failed
- Payment Pending Reminder
