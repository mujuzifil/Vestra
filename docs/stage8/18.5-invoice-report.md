# Stage 18.5 — Invoice Report

## Invoice Generation

Invoices are generated server-side using `barryvdh/laravel-dompdf`:

- Controller: `backend/app/Http/Controllers/Api/V1/InvoiceController.php`
- Service: `backend/app/Services/InvoicePdfService.php`
- View: `backend/resources/views/invoices/pdf.blade.php`

## Invoice Layout

| Section | Contents |
|---------|----------|
| Header | Company logo, name, address, email, phone |
| Invoice Meta | Invoice number, date, payment status, order status |
| Bill To | Customer name, email, address |
| Ship To | Recipient name, phone, address |
| Order Items | Product, SKU, quantity, unit price, line total |
| Totals | Subtotal, shipping, VAT/tax, grand total, amount paid, outstanding |
| Terms & Support | Support contacts and payment terms |
| Footer | Thank you message and company tagline |

## Configuration

Company details are read from `Setting` records:

| Setting Key | Purpose |
|-------------|---------|
| `company_name` | Invoice header name |
| `business_address` | Company address |
| `support_email` | Support email |
| `support_phone` | Support phone |
| `company_logo` | Logo image URL |

Fallbacks are provided for each value so invoices render even when settings are empty.

## Download

Customers download invoices from:

- Order detail page: "Download Invoice" button
- Direct API: `GET /api/v1/orders/{order}/invoice`

## Print

Because the response is a PDF, browser print dialogs can print the downloaded file directly.
