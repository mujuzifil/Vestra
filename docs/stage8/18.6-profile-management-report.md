# Stage 18.6 — Profile Management Report

## Objective

Allow customers to view and manage their personal profile, including name, email, phone, date of birth, gender, and profile photo.

## Implementation

### Backend

- Migration `add_profile_fields_to_users_table` added:
  - `first_name`, `last_name`, `date_of_birth`, `gender`, `avatar_path`, `preferences_json`.
- `User` model updated with new fillable fields, casts, and helpers:
  - `avatarUrl()` returns a public asset URL.
  - `preferences()` HasOne relation to `CustomerPreference`.
- `ProfileController` extended to support full profile updates with audit logging.
- New `AvatarController` handles upload and removal of profile photos.
- `UpdateProfileRequest` validates email uniqueness excluding the current user.
- `UploadAvatarRequest` validates image files (jpg/png/webp, max 2MB).

### Frontend

- New `/account/profile` page for editing profile fields.
- New `/account/profile/photo` page for avatar upload/preview/removal.
- Existing `/account/settings` page links to the new profile and photo pages.
- Dashboard welcome card displays avatar and profile completion.

### API Endpoints

| Method | Endpoint | Purpose |
|--------|----------|---------|
| GET | `/api/v1/auth/profile` | View profile |
| PUT | `/api/v1/auth/profile` | Update profile |
| POST | `/api/v1/auth/avatar` | Upload avatar |
| DELETE | `/api/v1/auth/avatar` | Remove avatar |

## Validation

- Email uniqueness enforced.
- Date of birth must be today or earlier.
- Gender limited to accepted values.
- Avatar image type and size validated.
