# Stage 18.4A.1 — SSL Validation Report

## Objective

Ensure `https://admin.vestradetergents.com` is served with a valid, trusted TLS certificate and that automatic renewal remains operational.

## Certificate Issuance

The production stack uses Let's Encrypt with the certbot container defined in `docker-compose.prod.yml`.

### Issue a certificate for the admin domain

Run on the production VPS:

```bash
cd /opt/vestra
docker compose -f docker-compose.prod.yml --env-file .env.production run --rm --entrypoint "" certbot \
  certbot certonly --webroot -w /var/www/certbot -d admin.vestradetergents.com \
  --agree-tos --non-interactive --email vestradetergument@gmail.com
```

This writes the certificate to `./certbot/conf/live/admin.vestradetergents.com/`.

### Reload nginx

After issuance:

```bash
docker compose -f docker-compose.prod.yml --env-file .env.production exec nginx nginx -s reload
```

## Validation Commands

### DNS resolution

```bash
nslookup admin.vestradetergents.com
dig admin.vestradetergents.com A
```

Expected: resolves to `187.77.84.119`.

### TLS handshake and certificate chain

```bash
echo | openssl s_client -connect admin.vestradetergents.com:443 -servername admin.vestradetergents.com 2>/dev/null | openssl x509 -noout -subject -issuer -dates
```

Expected output includes:

- `subject=CN = admin.vestradetergents.com`
- `issuer=C = US, O = Let's Encrypt, CN = ...`
- `notBefore` and `notAfter` showing a 90-day window

### Full TLS connection

```bash
curl -sI https://admin.vestradetergents.com/login
```

Expected: `HTTP/2 200` (or redirect to login for unauthenticated users) and no certificate warnings.

### Renewal dry-run

```bash
docker compose -f docker-compose.prod.yml --env-file .env.production run --rm --entrypoint "" certbot \
  certbot renew --dry-run
```

Expected: certbot reports that renewal is possible for all domains.

## Certificate Paths in nginx

The admin vhost is configured to load:

```nginx
ssl_certificate     /etc/letsencrypt/live/${ADMIN_DOMAIN}/fullchain.pem;
ssl_certificate_key /etc/letsencrypt/live/${ADMIN_DOMAIN}/privkey.pem;
ssl_trusted_certificate /etc/letsencrypt/live/${ADMIN_DOMAIN}/chain.pem;
```

These paths are mounted from `./certbot/conf` on the host.

## HTTPS Enforcement

- The default vhost on port 80 returns `301 https://$host$request_uri` for all requests.
- The admin vhost on port 443 emits HSTS: `Strict-Transport-Security: max-age=31536000; includeSubDomains; preload`.
