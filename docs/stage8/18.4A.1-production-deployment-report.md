# Stage 18.4A.1 — Production Deployment Report

## Deployment Overview

This stage corrects the legacy `/admin/*` redirect behaviour so that old bookmarks and search-indexed URLs continue to work after Filament was moved to its own subdomain with `->path('')`.

Only one infrastructure file is changed:

- `nginx/conf.d/vestra.conf.template`

## Environment Verification

Before deployment, confirm `.env.production` contains:

```bash
ADMIN_DOMAIN=admin.vestradetergents.com
```

This value is consumed by:

- `docker-compose.prod.yml` (nginx container environment)
- `backend/config/app.php` (`admin_domain` config)
- `backend/app/Providers/Filament/AdminPanelProvider.php` (`->domain(...)`)
- `scripts/deploy.sh` (pre-flight validation)

## Deployment Steps (Run on Production VPS)

### 1. Pull the updated configuration

From the deployment root (`/opt/vestra`):

```bash
cd /opt/vestra
git pull origin main
```

### 2. Validate the compose configuration

```bash
docker compose -f docker-compose.prod.yml --env-file .env.production config --quiet
```

A successful run returns no output.

### 3. Run the deployment script

```bash
./scripts/deploy.sh --build
```

Or, if a tagged image is already built and published:

```bash
./scripts/deploy.sh <image-tag>
```

The deploy script will:

- Validate required environment variables including `ADMIN_DOMAIN`.
- Take a pre-deployment backup.
- Run migrations.
- Start/restart services.
- Verify backend and frontend health.

### 4. Rebuild Laravel caches

After deployment, clear and rebuild caches so the new Filament domain/path configuration is active:

```bash
docker compose -f docker-compose.prod.yml --env-file .env.production exec backend php artisan config:cache
docker compose -f docker-compose.prod.yml --env-file .env.production exec backend php artisan route:cache
docker compose -f docker-compose.prod.yml --env-file .env.production exec backend php artisan view:cache
```

### 5. Reload nginx

```bash
docker compose -f docker-compose.prod.yml --env-file .env.production exec nginx nginx -s reload
```

### 6. Verify service health

```bash
docker compose -f docker-compose.prod.yml --env-file .env.production ps
docker compose -f docker-compose.prod.yml --env-file .env.production exec backend curl -fsS http://localhost:8080/api/v1/health
docker compose -f docker-compose.prod.yml --env-file .env.production exec frontend wget --no-verbose --tries=1 --spider http://localhost:3000/api/health
```

## Files Modified

| File | Change |
|------|--------|
| `nginx/conf.d/vestra.conf.template` | Replaced two literal `/admin` redirect locations with a single regex location that strips the `/admin` prefix and preserves the remainder of the path and query string. |

## Rollback

If a regression is detected, run the rollback script recorded by `deploy.sh`:

```bash
./scripts/rollback.sh
```

Then reload nginx:

```bash
docker compose -f docker-compose.prod.yml --env-file .env.production exec nginx nginx -s reload
```
