# Stage 18.4 — Admin Payment Report

## Admin Order Management

The Filament `OrderResource` provides the primary admin interface for payment visibility.

### Orders List

Columns displayed:

- Invoice number
- Customer name and email
- Item count
- Total amount
- Payment status badge
- Order status badge
- Created / updated timestamps

Filters available:

- Invoice / customer search
- Order status
- Payment status
- Date range
- Order value range
- Recently updated
- High value orders

### Order Detail View

The order detail page (`/admin/orders/{id}`) displays a dedicated **Payment Information** section:

- Payment method
- Payment status badge
- Amount paid
- Outstanding balance
- Transaction reference
- Payment date

It also displays:

- Order summary and totals
- Customer information
- Shipping address
- Ordered items
- Shipping and fulfilment details
- Order timeline
- Audit history

## Manual Status Actions

Admins can transition orders through the fulfilment lifecycle:

| Action | Visible When | Effect |
|--------|--------------|--------|
| Mark Paid | status = pending | Sets order status to `paid` and payment_status to `paid` |
| Mark Processing | status = paid | Sets order status to `processing` |
| Mark Packed | status = processing | Sets order status to `packed` |
| Mark Shipped | status = packed | Sets order status to `shipped` |
| Mark Delivered | status = shipped | Sets order status to `delivered` |
| Cancel Order | status in pending/processing/packed | Cancels order and restores stock |
| Refund Order | status in paid/processing/delivered | Refunds order and restores stock |

## Payment Transaction Resource

A dedicated `PaymentTransactionResource` in Filament is **not** implemented in this stage. All payment information required for operations is already accessible through the order detail view. If transaction-level search, export, or bulk operations become necessary, a dedicated resource can be added in a future stage.

## Audit Trail

All manual status transitions are logged via `AuditService` and appear in the order detail **Audit History** section.
