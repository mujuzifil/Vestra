# Stage 18.4A.1 — Stage Assessment

## Objective

Correct legacy `/admin/*` redirects so that bookmarks and search-indexed URLs for the old Filament admin path continue to work after the admin portal migrated to its own subdomain with Filament configured at the domain root (`->path('')`).

## What Was Changed

- `nginx/conf.d/vestra.conf.template`
  - Replaced two literal `/admin` redirect locations with a single regex location.
  - The new redirect strips the `/admin` prefix and preserves the remainder of the path and query string.

## What Was Verified

- The regex redirect correctly maps:
  - `/admin` → `https://admin.vestradetergents.com`
  - `/admin/login` → `https://admin.vestradetergents.com/login`
  - `/admin/orders` → `https://admin.vestradetergents.com/orders`
  - `/admin/products?filter=...` → `https://admin.vestradetergents.com/products?filter=...`
- No redirect loops are introduced.
- No `/admin` segment remains in redirected URLs.
- `ADMIN_DOMAIN` is already consumed by `docker-compose.prod.yml`, `backend/config/app.php`, `AdminPanelProvider.php`, and `scripts/deploy.sh`.

## Production Deployment Commands

Run on the VPS:

```bash
cd /opt/vestra
git pull origin main
docker compose -f docker-compose.prod.yml --env-file .env.production config --quiet
./scripts/deploy.sh --build

# Rebuild caches
docker compose -f docker-compose.prod.yml --env-file .env.production exec backend php artisan config:cache
docker compose -f docker-compose.prod.yml --env-file .env.production exec backend php artisan route:cache
docker compose -f docker-compose.prod.yml --env-file .env.production exec backend php artisan view:cache

# Reload nginx
docker compose -f docker-compose.prod.yml --env-file .env.production exec nginx nginx -s reload
```

## SSL Issuance Command

```bash
docker compose -f docker-compose.prod.yml --env-file .env.production run --rm --entrypoint "" certbot \
  certbot certonly --webroot -w /var/www/certbot -d admin.vestradetergents.com \
  --agree-tos --non-interactive --email vestradetergents@gmail.com

docker compose -f docker-compose.prod.yml --env-file .env.production exec nginx nginx -s reload
```
