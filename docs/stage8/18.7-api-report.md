## Objective

Document the complete REST API surface for the distributor portal, including authentication requirements and request/response conventions.

## Authentication & Middleware

All distributor portal endpoints require `auth:sanctum`. The portal group additionally passes through the custom `EnsureDistributor` middleware, which verifies that the authenticated user has the `distributor` role and an active distributor account. Non-distributors receive `403 Forbidden` with the message "Distributor access required."

The public application form (`POST /api/v1/distributor`) is unauthenticated but rate-limited by `throttle:distributor`.

## Request/Response Conventions

- All endpoints return a JSON wrapper: `{ "success": boolean, "data": ..., "message": string }`.
- Validation errors return `422 Unprocessable Entity` with Laravel's standard error bag.
- Resource collections are wrapped inside the `data` key.
- Monetary amounts are returned as formatted strings or decimals depending on the resource.
- Upload endpoints accept `multipart/form-data`; other endpoints accept and return JSON.
