# Stage 18.7 — Wholesale Ordering Report

## Objective

Enable distributors to browse a wholesale product catalog and place bulk orders through the existing cart and checkout infrastructure, with distributor-specific pricing and a choice of credit or Flutterwave payment.

## Distributor Pricing Service

The `App\Services\DistributorPriceService` resolves the best available unit price using a three-tier cascade:

1. **Negotiated price** — `DistributorProductPrice` record for the specific distributor that is currently effective (`effective_from` <= today <= `effective_until`).
2. **Volume tier price** — `DistributorPriceTier` record whose `min_quantity` / `max_quantity` range contains the requested quantity.
3. **Base distributor price** — the `distributor_price` column on the product.

If none of these are defined, the service falls back to the retail price via `resolveOrRetail()`. This ensures distributors always see a price while protecting configured margins.

## Wholesale Product Catalog

`App\Http\Controllers\Api\V1\Distributor\ProductController` exposes the catalog at `GET /api/v1/distributor/products`. Features include:

- Pagination (default 12 items, configurable up to 100 via `per_page`).
- Category filtering via `category`.
- Search by product name or SKU.
- Only active products are returned.
- Each product is decorated with `wholesale_price` for quantity 1 and includes loaded `distributorPriceTiers`.

Single-product detail is available at `GET /api/v1/distributor/products/{slug}`.

## Bulk Ordering via Cart + Distributor Checkout Channel

Distributor orders reuse the existing cart system. When ready to checkout, the frontend calls `POST /api/v1/checkout` with:

- `channel` set to `distributor`
- `distributor_branch_id` (optional)
- `payment_method` set to `credit` or a Flutterwave-supported method
- `shipping_address`

`CheckoutController` routes distributor-channel requests to `DistributorOrderService::createOrder`, which:

1. Loads the user's cart items.
2. Validates product status and available stock.
3. Resolves distributor pricing per line item using `DistributorPriceService`.
4. Computes subtotal, tax (from the `tax_rate` setting, default 18%), and total.
5. Creates an `Order` with `channel = distributor`, `distributor_id`, and a generated `VST-D-YYYYMMDD-####` invoice number.
6. Creates order items and clears the cart.

## Credit vs Flutterwave Payment

- **Credit:** If `payment_method` is `credit`, `CreditAccountService::authorize()` checks that the account is active and that `availableCredit()` covers the order total. On success, an `authorization` transaction is recorded and the order returns `payment_required: true`. The balance is captured when the order is fulfilled.
- **Flutterwave:** For other methods, `PaymentService::initiate()` generates a payment link and the order returns `payment_url`.

## Order Visibility

Distributors can list and view their own orders at `GET /api/v1/distributor/orders` and `GET /api/v1/distributor/orders/{order}`. The `DistributorOrderService` scopes queries by `distributor_id` and the `OrderPolicy` enforces ownership.
