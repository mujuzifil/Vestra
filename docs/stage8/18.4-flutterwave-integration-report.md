# Stage 18.4 — Flutterwave Integration Report

## Overview

The production payment workflow connects Stage 18.3 orders to Flutterwave. The backend integration was already largely complete; this stage fixed the production configuration defect and wired the frontend into the existing backend services.

## Configuration

| Environment Variable | Purpose | Location |
|----------------------|---------|----------|
| `FLUTTERWAVE_PUBLIC_KEY` | Public key for client-side configuration (reserved for future use) | `.env` |
| `FLUTTERWAVE_SECRET_KEY` | Server-side API secret for `/payments` and `/transactions/verify_by_reference` | `.env` |
| `FLUTTERWAVE_ENCRYPTION_KEY` | Reserved for future Flutterwave encryption features | `.env` |
| `FLUTTERWAVE_WEBHOOK_SECRET` | HMAC secret used to validate webhook signatures | `.env` |
| `FRONTEND_URL` | Base URL used to build the customer redirect back from Flutterwave | `.env` |

## Configuration Fix

`PaymentService::initiate()` builds the redirect URL with `config('app.frontend_url')`, but `backend/config/app.php` did not expose that key. The following entry was added:

```php
'frontend_url' => env('FRONTEND_URL', env('APP_URL', 'http://localhost')),
```

This ensures `config('app.frontend_url')` resolves correctly after `php artisan config:cache`.

## Backend Components

| Component | Responsibility |
|-----------|----------------|
| `PaymentService` | Creates transactions, initiates Flutterwave payments, verifies and processes payments, updates order/payment status, decrements stock idempotently |
| `FlutterwaveGateway` | Calls Flutterwave `/payments` and `/transactions/verify_by_reference`; validates webhook HMAC signatures |
| `PaymentController` | Exposes `initiate`, `verify`, `callback`, and `getTransaction` endpoints |
| `PaymentCallbackRequest` | Validates webhook payload and verifies `verif-hash` signature before allowing `callback` |
| `PaymentTransaction` model | Stores transaction state, references, amounts, and provider responses |

## API Endpoints

| Method | Endpoint | Auth | Purpose |
|--------|----------|------|---------|
| POST | `/api/v1/payments/initiate` | Yes | Create a Flutterwave transaction and return `payment_link` |
| GET | `/api/v1/payments/{reference}/verify` | Yes | Query Flutterwave and update order/payment status |
| GET | `/api/v1/payments/{reference}` | Yes | Return local transaction summary (status, amount, paid_at, order_id) |
| POST | `/api/v1/payments/callback` | No (signature) | Flutterwave webhook endpoint |

## Frontend Components

| Component | Purpose |
|-----------|---------|
| `frontend/lib/api/payments.ts` | API wrappers for `initiatePayment`, `verifyPayment`, and `getTransaction` |
| `frontend/app/checkout/confirm/confirm-page-client.tsx` | Enables "Proceed to Payment" and redirects to Flutterwave |
| `frontend/app/checkout/return/payment-return-client.tsx` | Handles Flutterwave redirect states and verifies payment |
| `frontend/app/account/orders/[id]/order-detail-client.tsx` | Allows customers to retry payment from order history |
