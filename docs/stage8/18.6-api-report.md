# Stage 18.6 — API Report

## Objective

Provide a complete REST API for the customer self-service portal.

## Endpoints

### Profile & Avatar

| Method | Endpoint | Auth | Description |
|--------|----------|------|-------------|
| GET | `/api/v1/auth/profile` | Yes | View customer profile |
| PUT | `/api/v1/auth/profile` | Yes | Update profile |
| POST | `/api/v1/auth/avatar` | Yes | Upload avatar |
| DELETE | `/api/v1/auth/avatar` | Yes | Remove avatar |

### Security

| Method | Endpoint | Auth | Description |
|--------|----------|------|-------------|
| POST | `/api/v1/auth/change-password` | Yes | Change password |
| GET | `/api/v1/auth/activity` | Yes | Activity timeline |

### Addresses

| Method | Endpoint | Auth | Description |
|--------|----------|------|-------------|
| GET | `/api/v1/auth/addresses` | Yes | List addresses |
| POST | `/api/v1/auth/addresses` | Yes | Create address |
| GET | `/api/v1/auth/addresses/{address}` | Yes | View address |
| PUT | `/api/v1/auth/addresses/{address}` | Yes | Update address |
| DELETE | `/api/v1/auth/addresses/{address}` | Yes | Delete address |

### Preferences

|--------|----------|------|-------------|

### Account Deletion

| Method | Endpoint | Auth | Description |
|--------|----------|------|-------------|
| POST | `/api/v1/auth/account-deletion-request` | Yes | Request deletion |

## Resources

- `CustomerResource` — customer profile shape.
- `AddressResource` — address shape.
- `PreferenceResource` — preferences shape.
- `ActivityResource` — audit log activity shape.

## Security

- All endpoints require `auth:sanctum`.
- Address endpoints use `CustomerAddressPolicy` for ownership.
- Change password endpoint has rate limiting.
