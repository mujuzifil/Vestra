# Stage 18.5 — Tracking Report

## Order Timeline States

| State | Icon | Meaning |
|-------|------|---------|
| Order Placed | Clock | Order created, awaiting payment or confirmation |
| Payment Received | CheckCircle | Payment confirmed |
| Preparing / Processing | Package | Order is being prepared |
| Packed | Box | Order packed, ready for dispatch |
| Ready for Dispatch | Truck | Order awaiting courier pickup |
| Dispatched / Shipped | TruckIcon | Order is in transit |
| In Transit | TruckIcon | Moving toward customer |
| Delivered | CheckCircle | Order delivered |
| Cancelled | XCircle | Order cancelled |
| Refunded | RotateCcw | Order refunded |

## Backend Timeline Generation

`Order::timeline()` builds events from:

1. Order creation
2. Payment received (when `payment_status = paid`)
3. `OrderStatusHistory` records (status transitions)
4. Dispatch event (when `dispatched_at` set)
5. Delivery event (when `delivered_at` set)

Events are sorted chronologically.

## Customer Tracking Experience

### Order Detail Page

- Visual timeline with icons, colors, timestamps, and actor.
- Tracking sidebar shows courier, tracking number, dispatch date, and estimated delivery.
- "Track Order" button links to `/track?invoice={invoice}`.

### Dedicated Tracking Page

URL: `/track`

- Search by invoice number or order ID.
- Authenticated customers see only their own orders.
- Displays:
  - Current status
  - Estimated delivery
  - Full shipment timeline
  - Tracking details
  - Shipping address
  - Link to full order details

## Estimated Delivery

- If dispatched: `dispatched_at + 3 days`
- If delivered: actual `delivered_at`
- Otherwise: `created_at + 7 days`
