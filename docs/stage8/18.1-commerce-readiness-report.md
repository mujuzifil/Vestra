# Stage 18.1 — Commerce Readiness Report

## Current State

### Existing commerce infrastructure
- Cart context and API (`frontend/lib/cart-context.tsx`, `frontend/lib/api/cart.ts`)
- Cart drawer UI (`frontend/components/cart/cart-drawer.tsx`)
- Checkout page with shipping, payment, review steps (`frontend/app/checkout/`)
- Order confirmation page (`frontend/app/checkout/confirm/`)
- Order history and tracking (`frontend/app/account/orders/`)
- Product catalogue with categories, images, pricing, stock quantity

### Current gaps
- Product pages did not expose Add to Cart / Buy Now actions.
- No quantity selector on product pages.
- Distributor CTA interrupted the purchase journey.
- No dedicated bulk-order quotation entry point.

## Changes Made in Stage 18.1
- Added quantity selector and commerce actions to product detail pages.
- Added quick Add to Cart buttons on product cards (home + products listing).
- Separated navigation journeys for retail, bulk, and distributor.
- Created `/bulk-orders` quotation request page.
- Added distributor success page with reference number.

## Reserved Future Work

- Persist guest cart reliably across sessions.
- Add cart page (full-page view in addition to drawer).
- Cart-level promotions and coupon input.
- Saved-for-later functionality.

### Checkout
- Shipping cost calculation.

- Multiple shipping addresses.

### Payments
- Flutterwave integration is already partially wired for digital payments.
- Complete webhook handling for payment status updates.
- Refund and failed-payment retry flows.

### Stock Availability
- Real-time stock checks before Add to Cart.
- Low-stock and back-in-stock notifications.

### Promotions & Coupons
- Coupon code UI in cart and checkout.
- Backend promotion engine.
- Volume discounts for bulk orders.

## Dependencies for Next Stage
- Cart API endpoints are already available.
- Checkout API endpoint is already available.
- Payment gateway (Flutterwave) is already configured.
- Stage 18.2 can proceed without new backend dependencies.

## Conclusion
The commerce foundation is established. Product pages now support the future purchase stages, and the three customer journeys are independent.
