# Stage 18.6 — Security Report

## Objective

Provide customers with account security visibility and secure password management.

## Implementation

### Password Change

- Existing `ChangePasswordController` already requires current password, validates new password, invalidates all other Sanctum tokens, and logs the change via `AuditService`.
- Frontend `/account/password` and `/account/settings` password tab provide the change password form.

### Security Information

- New `/account/security` page displays:
  - Last login timestamp.
  - Last password change timestamp (from audit logs).
  - Account creation date.
  - Security recommendations.
- Future placeholder for active devices/session list.

### API Endpoints

| Method | Endpoint | Purpose |
|--------|----------|---------|
| POST | `/api/v1/auth/change-password` | Change password |
| GET | `/api/v1/auth/activity` | View security/activity history |

## Security Controls

- Current password required before changing password.
- All other access tokens revoked on password change.
- Rate limiting on `/auth/change-password`.
- CSRF protection via Sanctum.
- Audit logs record password changes, profile updates, and address changes.
