# Stage 18.4A — Admin Migration Report

## Filament Configuration Changes

### `backend/app/Providers/Filament/AdminPanelProvider.php`

Before:

```php
return $panel
    ->default()
    ->id('admin')
    ->path('admin')
    ->login()
    ...
```

After:

```php
return $panel
    ->default()
    ->id('admin')
    ->domain(config('app.admin_domain'))
    ->path('')
    ->login()
    ...
```

### `backend/config/app.php`

Added:

```php
'admin_domain' => env('ADMIN_DOMAIN', ''),
```

## Effect

- Filament now generates URLs on `admin.vestradetergents.com` instead of `api.vestradetergents.com/admin`.
- The admin panel is served from the domain root (`/`), so `https://admin.vestradetergents.com` loads the dashboard directly.
- All named routes (`filament.admin.*`) continue to work; they now include the admin domain.

## Hardcoded URL Fix

`backend/app/Livewire/Admin/GlobalSearchCommandPalette.php` previously used hardcoded `/admin/...` URLs for placeholder search results. These were replaced with Filament route helpers:

- `route('filament.admin.resources.orders.edit', ['record' => 1])`
- `route('filament.admin.resources.products.edit', ['record' => 1])`
- `route('filament.admin.resources.customers.edit', ['record' => 1])`

## Redirects

Old URLs under the API domain redirect to the new admin domain:

```nginx
location /admin {
    return 301 https://${ADMIN_DOMAIN}$request_uri;
}

location /admin/ {
    return 301 https://${ADMIN_DOMAIN}$request_uri;
}
```

Because Filament path changed from `/admin` to `/`, the redirect preserves the full request URI. Existing bookmarks to `https://api.vestradetergents.com/admin/login` will redirect to `https://admin.vestradetergents.com/admin/login`, which will 404 because the path no longer includes `/admin`. Users should navigate to `https://admin.vestradetergents.com/login` directly.

## Authentication

- Admin authentication continues to use Laravel sessions backed by Redis.
- `SESSION_DOMAIN` remains `.vestradetergents.com`, so the session cookie is valid across all subdomains.
- `SESSION_SECURE_COOKIE=true` and `SESSION_SAME_SITE=strict` remain in effect.
- `EnsureAdminPasswordChanged` middleware uses route names for redirects, so it adapts automatically to the new domain.

## Post-Deploy Cache Clear

After deployment, clear Laravel caches so the new `ADMIN_DOMAIN` value is picked up:

```bash
docker compose -f docker-compose.prod.yml --env-file .env.production exec backend php artisan config:cache
docker compose -f docker-compose.prod.yml --env-file .env.production exec backend php artisan route:cache
docker compose -f docker-compose.prod.yml --env-file .env.production exec backend php artisan view:cache
```
