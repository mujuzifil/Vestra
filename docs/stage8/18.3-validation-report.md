- ✅ Guest cart merged before order creation.

## Edge Cases
- ✅ Out-of-stock products rejected by backend.
- ✅ Unavailable products rejected by backend.
- ✅ Price manipulation prevented by server-side totals.
- ✅ Missing address prevents submission.
- ✅ Duplicate submission prevented by loading state.
- ✅ Existing email during guest checkout prompts login.

## Performance
- Guest info persisted in localStorage to survive refresh.
- Order creation is a single API call for authenticated users.
- Guest checkout requires register + merge + checkout; handled sequentially with loading state.

## Known Limitations
- Guest account password is auto-generated; guest must use password reset to log in later.
- Pickup shipping method is disabled (placeholder).
- Payment redirect is not implemented (Stage 18.4).
