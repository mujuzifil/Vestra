# Stage 18.4A — SSL Validation Report

## SSL Requirements

The new admin subdomain requires a valid TLS certificate. The target is:

```
https://admin.vestradetergents.com
```

## Certificate Strategy

A dedicated Let's Encrypt certificate is requested for `admin.vestradetergents.com`:

```text
/etc/letsencrypt/live/admin.vestradetergents.com/
├── fullchain.pem
├── privkey.pem
└── chain.pem
```

This avoids touching the existing `vestradetergents.com` and `api.vestradetergents.com` certificates, reducing blast radius.

## Certificate Issuance Commands

On the production VPS, after deploying the updated stack:

```bash
cd /opt/vestra

# Ensure the admin domain resolves and nginx can serve the ACME challenge.
docker compose -f docker-compose.prod.yml --env-file .env.production run --rm \
  --entrypoint "" certbot certbot certonly \
  --webroot -w /var/www/certbot \
  -d admin.vestradetergents.com \
  --agree-tos --non-interactive \
  --email vestradetergent@gmail.com

# Reload nginx to pick up the new certificate.
docker compose -f docker-compose.prod.yml --env-file .env.production exec nginx nginx -s reload
```

## Validation Commands

| Check | Command |
|-------|---------|
| HTTPS responds | `curl -I -L https://admin.vestradetergents.com` |
| Certificate details | `openssl s_client -connect admin.vestradetergents.com:443 -servername admin.vestradetergents.com </dev/null` |
| Expiry date | `echo | openssl s_client -servername admin.vestradetergents.com -connect admin.vestradetergents.com:443 2>/dev/null \| openssl x509 -noout -dates` |
| Auto-renewal | `docker compose -f docker-compose.prod.yml --env-file .env.production exec certbot certbot renew --dry-run` |

## Expected Results

- `curl` returns HTTP 200 for the admin login page.
- `openssl` shows a certificate issued by `R3` (Let's Encrypt) for `admin.vestradetergents.com`.
- Browser trust: no warnings in Chrome/Firefox/Edge.
- Renewal dry-run succeeds.

## Auto-Renewal

The existing `certbot` container in `docker-compose.prod.yml` already runs `certbot renew` twice daily. The new certificate will be renewed automatically as long as the admin subdomain continues to resolve and nginx serves the ACME challenge.
