# Stage 18.6 — Address Book Report

## Objective

Provide customers with a complete address book supporting multiple addresses, default shipping/billing flags, and checkout auto-fill.

## Implementation

### Backend

- Migration `enhance_customer_addresses_table` added:
  - `address_line_2`, `postal_code`, `country`, `delivery_notes`, `is_default_shipping`, `is_default_billing`.
- `CustomerAddress` model updated with new fillable fields, casts, and helper scopes.
- `AddressController` updated to manage default shipping/billing flags and ensure only one default of each type per user.
- `StoreAddressRequest` validates all new fields.
- `AddressResource` exposes all new fields.
- `CustomerAddressPolicy` enforces ownership.

### Frontend

- `/account/addresses` enhanced with new fields and separate Default Shipping / Default Billing toggles.
- Address cards display badges for default shipping/billing.
- Checkout pre-fills customer information and default shipping/billing addresses.

### API Endpoints

| Method | Endpoint | Purpose |
|--------|----------|---------|
| GET | `/api/v1/auth/addresses` | List addresses |
| POST | `/api/v1/auth/addresses` | Create address |
| GET | `/api/v1/auth/addresses/{address}` | View address |
| PUT | `/api/v1/auth/addresses/{address}` | Update address |
| DELETE | `/api/v1/auth/addresses/{address}` | Delete address |

## Validation

- Address ownership enforced via policy.
- Only one default shipping and one default billing address per customer.
- Required fields validated (label, full_name, phone, city, address_line).
