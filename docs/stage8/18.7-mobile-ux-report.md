# Stage 18.7 — Mobile UX Report

## Objective

Ensure the distributor portal is usable on phones and tablets through responsive layouts, a mobile navigation drawer, adequate touch targets, and accessible markup.

## Responsive Layout

The portal shell in `frontend/app/distributor/layout.tsx` uses a flex layout with a sticky sidebar on large screens (`lg:flex w-64`) and a full-width main content area. On mobile the sidebar is hidden and replaced by a drawer launched from a fixed top-left toggle button.

Page content consistently uses Tailwind responsive prefixes:

- Stat cards: `grid-cols-2 lg:grid-cols-4`
- Branch and invoice grids: `sm:grid-cols-2 lg:grid-cols-3`
- Forms: `grid sm:grid-cols-2 gap-5`
- Tables: wrapped in `overflow-x-auto` so wide content scrolls horizontally without breaking layout.

This approach keeps dashboards readable on small screens while taking advantage of larger monitors. Containers use consistent padding (`p-4 sm:p-6 lg:p-8`) so content never touches screen edges, and card components use soft shadows and rounded corners to maintain visual hierarchy on smaller viewports.

## Mobile Drawer

`frontend/components/distributor/distributor-sidebar.tsx` implements the navigation:

- A hamburger button fixed at the top-left on mobile (`lg:hidden`) opens the drawer.
- The drawer slides in from the left over a dark backdrop and closes on backdrop click or navigation.
- A close button is provided inside the drawer with an `aria-label="Close menu"`.
- The same navigation content is reused for desktop and mobile to keep links consistent.

## Touch Targets

Interactive elements use generous padding to meet minimum touch target recommendations:

- Sidebar links: `px-3 py-2.5`
- Action buttons: `px-4 py-2.5` to `px-6 py-3`
- Form file drop zone: full-width padded area
- Small icon buttons inside cards: `px-3 py-1.5`

Buttons are clearly separated and use hover/focus states for visual feedback. Form fields stack vertically on small screens, with each input taking the full width and clear error text below the field. Long text inputs and textareas remain usable without horizontal scrolling.

## Accessibility Notes

- `aria-label` attributes are present on the mobile menu open/close buttons.
- `aria-current="page"` marks the active navigation item.
- Section headings use `aria-labelledby` on public pages.
- Form inputs are associated with labels.
- Focus rings use Tailwind's `focus:ring-green-500` on checkboxes and inputs.
- Loading states display centered spinners with appropriate contrast.

## Component Patterns

Reusable components keep the mobile experience consistent:

- `DistributorStatCard` — compact summary cards that adapt to grid columns.
- `EmptyState` — centered message with icon for empty lists.
- `form-field` inputs — full-width inputs with error text.

## Performance & Contrast

Page content is rendered on the client through Next.js client components, with loading spinners displayed while React Query fetches data. Color contrast follows the existing design system: dark headings (`text-[#0a1628]`) on light backgrounds (`bg-[#f8fafc]`) and green action colors (`bg-green-600`) on white cards. Status badges use distinct background colors so distributors can scan order and invoice states quickly.
